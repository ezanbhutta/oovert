# Case study — what to send me

This is the shopping list for one case study. The template
(`work/case-study.html`) is already built with every animation and transition
wired up; it just needs your real images and a few lines of copy dropped in.

You don't have to fill every section — skip any that don't apply and I'll
remove them cleanly. But the order below is the intended narrative arc.

## How it works

- Every image slot is a **placeholder** that already states its ratio, size and
  purpose on screen. To go live, each one is swapped for a single `<img>` — the
  exact tag is written in an HTML comment right above the placeholder.
- Copy is written as `[ bracketed guidance ]`. Replace the brackets with your
  words; the guidance tells you the intended length and tone.
- Motion, smooth-scroll and the lightbox are automatic. Nothing you send needs
  to be animated — the sequencing does the work.

## Image specs (please read once)

- **Format:** JPG for photography/mockups, PNG or SVG for the logo (transparent).
  Send WebP too if you have it; otherwise I'll generate WebP on the way in.
- **Colour:** sRGB.
- **Size:** the numbers in each slot are the *minimum long edge*. Bigger is fine,
  I'll compress and generate responsive sizes. Aim for high-quality source.
- **Naming:** anything is fine, but `hero.jpg`, `intro.jpg`, `logo.png` … matches
  the comments in the file and makes the swap trivial.

## When you send them

Drop the images in a folder (or a zip) named for the project and tell me the
project name + scope. I'll create `work/<project>.html` from this template, wire
it into the Evidence section on the homepage as a clickable entry, optimize the
images, and verify the whole thing before it ships.
